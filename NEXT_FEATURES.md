# ClaraVision Next Features

## 当前结论

ClaraVision 已经从网页看板推进到桌面脑核形态。后续重点统一放在透明无背景版本：

- 透明脑核模式是主体验：适合放在桌面上，只显示活体脑核；桌面版会覆盖屏幕，脑核主体保持正圆。
- 完整模式只作为调试入口：用于查看 Memoria、Continuity、agent 来源和活动摘要。

下一阶段应该开始增加功能，但功能必须服务于“活体思维”这个主体验，不要再把页面做回 dashboard。

## 已完成：第一批视觉交互

- 悬停节点显示短摘要。
- 点击节点后聚焦并放大。
- 双击或 Esc 复位视角。
- O 切换完整/透明脑核模式。
- R 刷新静态快照。
- Electron 右键菜单支持刷新、复位、切换模式和退出。
- 透明脑核增加持续呼吸感。
- S 切换空闲 / 读取 / 思考状态。
- B 切换亮度。
- C 切换透明视觉档位：纯透明 / 柔光 / 高对比。
- + 切换脑核大小。
- P 暂停 / 继续动画。
- 点击记忆点会聚焦同来源记忆团。
- 透明桌面版固定覆盖整块屏幕，避免历史窗口位置把脑核挤到角落。
- Cmd/Ctrl + Shift + Space 显示 / 隐藏透明桌面版。
- Cmd/Ctrl + Shift + I 切换鼠标交互：自动点穿 / 始终可点 / 始终点穿。
- 私有记忆只作为受限暗粒子进入视觉密度，不显示正文。

## 已完成：第三批视觉主题 — 科幻全息风

### 1. 配色全换

暖金 → 冷白蓝全息（分支 `sci-fi-ui`，tag `v0.2-sci-fi-base`）。所有 agent、记忆、信号节点颜色统一改为 cyan/ice-blue 系列。颜色定义集中在 `AGENT_COLORS` + `colorFor()` + CSS 变量，便于后续添加第三主题。

### 2. 雷达扫描弧

脑核外圈持续旋转的扇形扫描弧，带衰减拖尾和亮边引导线。alpha 控制在 0.04，不抢主视觉。

### 3. 六边形框架

脑核外圈的六边形几何框架，缓慢旋转。alpha 0.04。

### 4. CSS 全息效果

- 扫描线 overlay（水平线条模拟 CRT/全息屏幕，alpha 0.005）
- 微暗角（边缘蓝色辉光）
- 全局文字辉光（text-shadow）
- mono 字体用于数据读数和按钮
- orb 对比度预设调亮（三个档位适配不同桌面背景）

### 5. 双分支架构

- `main`：暖金活体脑核
- `sci-fi-ui`：冷白蓝全息

两个分支 app.js 保持相同的渲染逻辑和参数，仅颜色值不同。后续添加新功能时两个分支同步更新。

## 已完成：第二批脑核能力

### 1. 三种状态动画

- 空闲：慢呼吸。
- 读取：外圈记忆向中心流动。
- 思考：中心变亮，信号分叉。

### 2. 记忆云团聚焦增强

点击记忆点时，同来源记忆团会变亮，其他区域会压暗。

### 3. 透明脑核窗口控制

透明脑核模式已经支持：

- 透明版不记住窗口大小和位置，保持整屏居中。
- 全局快捷键显示/隐藏。
- 右键菜单只保留对话、视觉档位、暂停、鼠标交互、清空会话、刷新、退出。

### 4. 透明度和大小调节

- 小 / 中 / 大。
- 低亮 / 标准 / 高亮。
- 纯透明 / 柔光 / 高对比。
- 是否置顶。

### 5. 低打扰常驻

- 透明桌面版默认自动点穿，不挡后面的桌面或应用。
- 鼠标靠近脑核、输入区或控制区时自动恢复交互。
- 输入框默认收成轻入口，主动输入、思考、执行或出错时才展开。
- 空闲时降低刷新负担，思考和执行时再提高活跃度。

## 优先级 1：接入运行状态

### 已完成（v0.4-agent-bridge）

#### 1. 对话状态联动

ClaraVision 内部对话的 status（thinking/executing/done/error）已映射到 brainState，按 S 切状态时有冲击波效果。

#### 2. Hermes hooks 事件流

通过 `claravision-bridge` Hermes 插件，实时将 Hermes 生命周期事件写入 `~/.hermes/claravision-events.jsonl`：
- `session_start` → 脑核进入"聆听"状态
- `llm_start` → 脑核进入"思考"状态
- `llm_end` → 脑核进入"完成"状态，2秒后回"空闲"
- `tool_start`（记忆类工具）→ 脑核进入"读取"状态
- `tool_start`（其他工具）→ 脑核进入"执行"状态
- `tool_end` → 脑核回到"思考"
- `session_end` → 脑核回"空闲"

ClaraVision main.js 用 fs.watch + 2秒轮询读取事件文件，通过 IPC 推给前端。

### 3. 语音状态

语音能力不是下一步立刻做，但要预留状态：

- 聆听：外圈收缩，出现声波。
- 理解：信号进入中心。
- 回答：中心向外释放脉冲。

麦克风权限、语音识别和语音输出都应放在 Electron 壳，不放在 Web 视觉层。

## 暂时不做

- 不做复杂聊天窗口。
- 不做完整 agent 执行器。
- 不让 ClaraVision 保存 Memoria 或 Continuity 数据。
- 不让 Memoria 和 Continuity 因 ClaraVision 产生运行时依赖。
- 不把所有数据都塞回列表。

3. Clara / Lara / Codex / Hermes 的实时状态接入。
