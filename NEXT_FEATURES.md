# ClaraVision Next Features

## 当前结论

ClaraVision 已经从网页看板推进到桌面脑核形态。后续重点统一放在透明无背景版本：

- 透明脑核模式是主体验：适合放在桌面上，只显示活体脑核；桌面版会覆盖屏幕，脑核主体保持正圆。
- 完整模式只作为调试入口：用于查看 Memoria、Continuity、agent 来源和活动摘要。

下一阶段应该开始增加功能，但功能必须服务于“活体思维”这个主体验，不要再把页面做回 dashboard。

## 已完成：第一批视觉交互

- 悬停节点显示短摘要。
- 点击节点后聚焦并放大。
- 双击或 Esc 复位视角。
- O 切换完整/透明脑核模式。
- R 刷新静态快照。
- Electron 右键菜单支持刷新、复位、切换模式和退出。
- 透明脑核增加持续呼吸感。
- S 切换空闲 / 读取 / 思考状态。
- B 切换亮度。
- C 切换透明视觉档位：纯透明 / 柔光 / 高对比。
- + 切换脑核大小。
- P 暂停 / 继续动画。
- 点击记忆点会聚焦同来源记忆团。
- 透明桌面版固定覆盖整块屏幕，避免历史窗口位置把脑核挤到角落。
- Cmd/Ctrl + Shift + Space 显示 / 隐藏透明桌面版。
- Cmd/Ctrl + Shift + I 切换鼠标交互：自动点穿 / 始终可点 / 始终点穿。
- 私有记忆只作为受限暗粒子进入视觉密度，不显示正文。

## 已完成：第二批脑核能力

### 1. 三种状态动画

- 空闲：慢呼吸。
- 读取：外圈记忆向中心流动。
- 思考：中心变亮，信号分叉。

### 2. 记忆云团聚焦增强

点击记忆点时，同来源记忆团会变亮，其他区域会压暗。

### 3. 透明脑核窗口控制

透明脑核模式已经支持：

- 透明版不记住窗口大小和位置，保持整屏居中。
- 全局快捷键显示/隐藏。
- 右键菜单只保留对话、视觉档位、暂停、鼠标交互、清空会话、刷新、退出。

### 4. 透明度和大小调节

- 小 / 中 / 大。
- 低亮 / 标准 / 高亮。
- 纯透明 / 柔光 / 高对比。
- 是否置顶。

### 5. 低打扰常驻

- 透明桌面版默认自动点穿，不挡后面的桌面或应用。
- 鼠标靠近脑核、输入区或控制区时自动恢复交互。
- 输入框默认收成轻入口，主动输入、思考、执行或出错时才展开。
- 空闲时降低刷新负担，思考和执行时再提高活跃度。

## 优先级 1：接入运行状态

### 1. agent 状态事件

后续可以把 Clara、Lara、Codex、Hermes 的状态变成事件：

- 正在读取记忆。
- 正在调用工具。
- 正在写回。
- 等待用户确认。
- 出错或失败。

这些事件先进入统一状态对象，再由脑核表现为光流、闪烁、颜色变化。

### 2. 语音状态

语音能力不是下一步立刻做，但要预留状态：

- 聆听：外圈收缩，出现声波。
- 理解：信号进入中心。
- 回答：中心向外释放脉冲。

麦克风权限、语音识别和语音输出都应放在 Electron 壳，不放在 Web 视觉层。

## 暂时不做

- 不做复杂聊天窗口。
- 不做完整 agent 执行器。
- 不让 ClaraVision 保存 Memoria 或 Continuity 数据。
- 不让 Memoria 和 Continuity 因 ClaraVision 产生运行时依赖。
- 不把所有数据都塞回列表。

## 下一轮建议

下一轮最值得做：

1. agent 真实运行状态事件。
2. 语音状态预留。
3. Clara / Lara / Codex / Hermes 的实时状态接入。
4. 讨论是否需要真正的常驻桌面客户端功能。
