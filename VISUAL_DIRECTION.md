# ClaraVision Visual Direction

## 目标

ClaraVision 下一阶段不再按“看板”思路推进，而是做成一个像活体思维一样的桌面可视化对象。

用户看到的第一屏应该是一个有呼吸、有能量流动、有记忆聚合的“脑核”，而不是一组数据卡片。数据仍然来自 Memoria 和 Continuity，但表现方式要从“列表展示”改成“状态显影”。

## 参考图要点

参考图：`/Users/zhouwei/Downloads/images.jpeg`

这张图值得学习的地方：

- 中心有高能核心，不是平均亮度。
- 外圈不是规则线框，而是粒子形成的旋臂和碎片轨迹。
- 信号像能量流，不像普通连接线。
- 背景足够暗，亮点才有空间感。
- 画面有局部爆亮和暗区，整体更像正在活动的意识体。
- HUD 线条只是辅助，不是主体。

当前 ClaraVision 的问题：

- 数据区太像 dashboard。
- 线条太直，信号缺少速度和生命感。
- 粒子密度和明暗层次不够。
- 视觉中心不够强，外圈没有旋涡和记忆云团。
- 文字信息太早出现，抢了主体画面。

## 产品形态判断

这不是 Web 的极限。Canvas/WebGL 足够先做出更好的活体视觉效果。

现在不应该直接进入 Electron 桌面版。否则只是把一个看板装进桌面窗口里。

正确顺序：

1. 先在 Web 里做出真正像活体思维的视觉原型。
2. 继续使用手动生成的 `data/state.json` 静态快照，保持启动快。
3. 视觉和交互确认后，再用 Electron 包成桌面应用。
4. 桌面版再接入常驻、置顶、快捷键、麦克风、语音和 agent 运行状态。

## 第一版活体模式

### 默认界面

- 第一屏只显示脑核主体。
- 数据面板默认隐藏或折叠。
- 只有鼠标靠近、点击、缩放进入局部时，才显示少量文字。
- 视觉主体必须占据页面中心，不再被卡片分散。

### 脑核结构

- 中心：ClaraCore 当前意识核心。
- 内圈：Continuity 当前继续位置。
- 中圈：Clara / Lara / Codex / Hermes 等 agent 信号源。
- 外圈：Memoria 记忆星云。
- 远场：环境粒子和低能背景噪声。

### 记忆表现

Memoria 的几百条记忆不要直接铺成列表。

应该按以下维度形成记忆云团：

- 来源 agent：Clara、Lara、Codex、Hermes。
- 时间新旧：新的更亮，旧的更暗。
- 当前相关性：与 Continuity 当前线有关的更靠近中心。
- 主题聚合：相似记忆形成局部星云。

### 状态表现

至少需要三种状态：

- 空闲：缓慢呼吸，粒子低速漂移。
- 读取：外圈记忆点被局部点亮，信号向内流入。
- 思考：中心变亮，信号分叉、绕行、汇聚。

后续桌面版再加：

- 聆听：外圈收缩，麦克风输入形成波纹。
- 回答：核心向外释放脉冲。
- 写回：一条高亮轨迹从中心回到 Memoria 或 Continuity 区域。

## 视觉验收标准

下一版不能只看“有没有功能”，要按观感验收：

- 第一眼不像 dashboard。
- 不需要读文字，也能感觉它在活动。
- 中心必须有明显生命感和呼吸感。
- 粒子必须有密度差、速度差、亮度差。
- 信号流动不能只是直线移动，要有弯曲、尾迹、分叉和衰减。
- 大屏下必须保持圆形脑核，不允许拉伸成扁圆。
- Memoria 数据多时，画面应该更丰富，而不是页面更拥挤。

## 数据边界

当前仍然保持静态快照模式：

```text
手动运行 build_state.py -> 生成 data/state.json -> 页面快速读取静态快照
```

ClaraVision 只展示，不保存事实，不替代 Memoria 或 Continuity，也不让两个系统因为 ClaraVision 产生直接依赖。

## 参考 jarvis-ai 后的取舍

`jarvis-ai` 的价值不在于直接搬视觉代码，而在于产品形态：

- 桌面应用
- 常驻窗口
- 热键唤醒
- 语音输入/输出
- 后台任务状态
- agent 状态事件

ClaraVision 应该借鉴这个方向，但不直接迁移它的整套运行时。

短期先做 Web 活体原型。中期用 Electron 包成桌面。长期再接语音、热键、常驻和 agent 状态流。

## 下一步实现建议

1. 新增“活体模式”布局，隐藏下方列表面板。
2. 重写 Canvas 粒子系统，让记忆形成旋涡星云。
3. 把信号线改成弯曲路径和粒子尾迹。
4. 增加空闲、读取、思考三种视觉状态。
5. 点击记忆团时只浮出少量中文摘要。
6. 保留当前静态快照数据流。
7. 活体模式验收通过后，再开始 Electron 桌面壳。
